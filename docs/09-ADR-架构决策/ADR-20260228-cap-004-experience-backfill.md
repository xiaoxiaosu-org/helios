# ADR-20260228: CAP-004 经验库补齐策略

## 状态
Accepted

## 日期
2026-02-28

## 背景
- CAP-004 已落地架构依赖门禁并通过校验，但经验沉淀初期使用了 PR 豁免。
- 协作要求明确提出：涉及经验库未补齐问题时，需要补齐经验库而非长期依赖豁免。

## 决策
- 在 `docs/02-架构/工程治理/经验库/` 新增 CAP-004 落地经验文档，记录实施步骤、踩坑与修复方式。
- 更新经验库索引，确保经验文档可发现、可复用。
- PR 口径从“豁免补齐”切换为“已补齐经验库”。

## 影响

### 正面影响
- 降低后续项目重复踩坑概率，形成可迁移实践资产。
- 使高风险治理变更更容易一次通过，减少豁免依赖。

### 负面影响
- 文档维护成本上升，需持续保持经验文档与实现一致。

## 备选方案
- 继续保留豁免，不补经验库：拒绝，不满足协作要求且会积累治理债务。
- 仅在 PR 描述中记录经验：拒绝，难以复用且不可系统检索。

## 版本影响
- 是否影响 API：否
- 是否影响事件模型：否
- 是否影响数据结构：否
