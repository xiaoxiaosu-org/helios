# ADR-20260228: CAP-007 DoD 口径同步到执行计划

## 状态
Accepted

## 日期
2026-02-28

## 背景
- `scripts/cap/CAP-007/verify.sh` 已将 CAP-007 验收强化为“关键字段逐项校验 + 每条 TD 状态合法性校验”。
- 执行计划中的 CAP-007 DoD 仍停留在“文档存在 + 至少一条 TD 记录”，与实际门禁口径不一致。

## 决策
- 更新 `docs/02-架构/执行计划/active/PLAN-20260227-01-工程智能化路线图.md` 的 CAP-007 DoD。
- 将关键字段要求（`Owner`、`最近更新`、`验收标准`）与状态合法性要求（`Open|In Progress|Blocked|Done`）纳入 DoD 描述。
- 保持验收入口不变，继续通过 `scripts/cap/verify.sh CAP-007` 机械验证。

## 影响

### 正面影响
- 文档口径与脚本实现一致，降低评审歧义与误判。
- CAP-007 验收标准可追溯，便于后续治理与审计。

### 负面影响
- 文档更新触发 ADR 维护成本上升，需要同步维护索引。

## 备选方案
- 仅修改脚本不更新执行计划 DoD：拒绝，会导致“实现与文档口径漂移”。
- 回退脚本到最小校验口径：拒绝，无法满足技术债治理要求。

## 版本影响
- 是否影响 API：否
- 是否影响事件模型：否
- 是否影响数据结构：否
