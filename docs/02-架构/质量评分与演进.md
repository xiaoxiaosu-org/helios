# 质量评分与演进（QUALITY_SCORE）

本文件用于把“工程质量与智能化程度”变成可量化、可审计、可持续改进的目标。

## 评分维度（初版）

维度建议先从“可执行/可验证/可演进”出发，逐步细化：
- 可执行性：关键流程是否都有脚本入口（而非只写在文档里）
- 可验证性：脚本是否产出可审计证据（日志/报告/截图/覆盖率等）
- 机械化程度：约束是否进入 CI/lints/结构测试（失败信息是否给出下一步修复指令）
- 可观测闭环：能否按 `traceId` 追到关键链路结果与耗时
- 熵控制：是否有周期性 doc-gardening / 偏差扫描与修复机制
