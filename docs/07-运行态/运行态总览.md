# 运行态总览

## 目的
维护 Trace 传播、上下文模型、窗口/缓存、回放与复盘机制。

## 最低要求
- 关键链路必须携带 traceId。
- 涉及流程时需关联 workflowInstanceId、stepId。
- 应提供最小可执行查询入口（见 `可观测查询剧本.md`），可按 traceId 回查 result 与 durationMs。

## 统一入口
- 工作项主文件：`docs/02-架构/执行计划/backlog.yaml`
- 命令聚合：`scripts/workflow/run.sh overview json`
- 本地看板：`scripts/workflow/run.sh overview serve 127.0.0.1 8787`
- 同步命令：`scripts/workflow/run.sh backlog build`
- 文档库校验：`scripts/docs/library-check.sh all`（支持模块化执行）
- 看板读取来源：`backlog.yaml`（管理视图）+ `artifacts/workflow/` + `git status`
